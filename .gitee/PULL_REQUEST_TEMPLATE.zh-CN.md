# PR 合入模板

**注：经过自检不涉及的可标注“不涉及”或直接打勾，特殊情况请文字备注。不符合规范的 PR 不允许合入，请（后备）commit 注意。**

---

## 1. 修改描述
- **修改原因：**  
- **修改内容：**  

---

## 2. 功能验证
- [ ] **功能自验**
- [ ] **本地自验用例截图**
- [ ] **冒烟是否通过** （填入群链接的自验证报告中，如未通过，请说明：____________________ ，功能代码请主动申报添加冒烟）

---

## 3. 代码检视
- **要求：**
  - 合入代码超过 200 行，需三人以上会议检视。
  - 检视密度≥1个/100行。
  - 检视缺陷密度未达要求需提供说明。
  - 大于 1000 行代码原则上不允许合入，需进行备案。
- [ ] **是否经过代码检视**
- [ ] **是否具备 UT 测试用例看护** （如不具备，请说明原因：____________________）

- **检视意见数：____ 条** （请填写本次检视的意见总数，用于commit合入前审视）

---

## 4. 安全自检

### Python、C++：
- [ ] **对外接口新增/删除/变更，是否需要更新外部输入表格**
- [ ] **不允许私有文件操作，是否使用公共函数**
- [ ] **数组使用是否校验越界场景**
- [ ] **是否对正则表达式做 ReDos 校验**
- [ ] **对除法操作是否进行除零校验**
- [ ] **是否充分校验接口返回值的异常情况**
- [ ] **是否充分校验接口输入值的异常情况**
- [ ] **日志是否避免暴露代码细节和敏感信息**

### C++：
- [ ] **指针使用前是否判空**
- [ ] **数值计算是否校验溢出和反转**
- [ ] **是否存在内存泄漏（异常场景下是否释放内存）**

---

## 5. 变更知会
- [ ] **资料修改**
- [ ] **变更通知（消息知会 + 邮件知会）**
