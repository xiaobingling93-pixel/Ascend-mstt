
# msprobe 工具精度数据采集指南

## 1. 专业名词解释

* **msprobe**：MindSpore 提供的用于精度数据采集的工具。
* **静态图**：在编译时就确定网络结构，提升执行效率。
* **动态图**：运行时动态构建网络，便于调试和开发。
* **L0、L1、L2 Level**：精度数据采集的不同等级，表示数据采集的粒度。
    * **L0 Level**：Cell 级数据采集。
    * **L1 Level**：API 级数据采集。
    * **L2 Level**：Kernel 级数据采集。
* **高阶 API**：如 `mindspore.train.Model`，封装了训练过程的高级接口。

## 2. 概述

msprobe 工具通过在训练脚本中添加 `dump` 接口并启动训练的方式，采集模型在运行过程中的精度数据。该工具支持不同场景和精度数据采集等级：

* **静态图场景**：仅支持 Kernel 级（L2 Level）数据采集。
* **动态图场景**：支持 Cell（L0 Level）、API（L1 Level）、Kernel（L2 Level）级数据采集。

> **注意**：动态图模式下，使用 `PSJit` 或 `PIJit` 装饰的部分实际以静态图模式执行，此时的 Kernel 级（L2 Level）数据采集方式与静态图场景相同。

## 3. 快速入门

以下通过一个简单的示例，展示如何在 MindSpore 中使用 msprobe 工具进行精度数据采集。

### 3.1 示例代码

**静态图场景：**

```python
import mindspore as ms
ms.set_context(mode=ms.GRAPH_MODE, device_target="Ascend")

from msprobe.mindspore import PrecisionDebugger
debugger = PrecisionDebugger(config_path="./config.json")
debugger.start()
# 这里添加模型的定义和训练代码
# 请勿将以上初始化流程置于模型实例化或 mindspore.communication.init 调用后
```

**动态图场景（使用高阶 API，非 L2 Level）：**

```python
import mindspore as ms
from mindspore.train import Model
ms.set_context(mode=ms.PYNATIVE_MODE, device_target="Ascend")

from msprobe.mindspore import PrecisionDebugger
from msprobe.mindspore.common.utils import MsprobeStep
debugger = PrecisionDebugger(config_path="./config.json")

# 模型和损失函数的定义
model = Network()
loss_fn = ...
optimizer = ...

trainer = Model(model, loss_fn=loss_fn, optimizer=optimizer)
trainer.train(1, train_dataset, callbacks=[MsprobeStep(debugger)])
```

## 4. 详细说明

### 4.1 精度数据采集等级

* **L0 Level（Cell 级）**：采集 `Cell` 对象的数据，适用于需要分析特定网络模块的情况。
* **L1 Level（API 级）**：采集 MindSpore API 的输入输出数据，适用于定位 API 层面的精度问题。
* **L2 Level（Kernel 级）**：采集底层算子的输入输出数据，适用于深入分析算子级别的精度问题。

### 4.2 动态图与静态图场景

* **静态图场景**：网络在编译时确定结构，msprobe 仅支持 L2 Level 的数据采集。
* **动态图场景**：网络在运行时动态构建，msprobe 支持 L0、L1、L2 Level 的数据采集。

> **提示**：当使用 `PSJit` 或 `PIJit` 装饰器时，装饰的部分以静态图方式执行，其 Kernel 级数据采集方式与静态图相同。

### 4.3 使用高阶 API 与否的区别

* **使用高阶 API（如 `Model`）**：需要使用 `MsprobeStep` 回调类来控制数据采集的启停，适用于 L0 和 L1 Level。
* **未使用高阶 API**：手动在训练循环中调用 `start`、`stop`、`step` 等接口，适用于所有 Level。

## 5. 接口介绍

### 5.1 PrecisionDebugger

**功能说明**：通过加载配置文件来确定数据采集的详细设置。

**使用方法：**

```python
from msprobe.mindspore import PrecisionDebugger
debugger = PrecisionDebugger(config_path="./config.json")
```

#### 5.1.1 start

**功能说明**：启动精度数据采集。

**原型：**

```python
start(model=None)
```

**参数说明：**

* `model`：可选，传入 `mindspore.nn.Cell` 实例。在 L0 Level 或需要采集 `primitive op` 的 L1 Level 时传入。

#### 5.1.2 stop

**功能说明**：停止数据采集。需要与 `start` 搭配使用。

**原型：**

```python
stop()
```

#### 5.1.3 step

**功能说明**：标记一个训练步骤的结束。

**原型：**

```python
step()
```

#### 5.1.4 forward_backward_dump_end

**功能说明**：标记前向和反向数据采集的结束。

**原型：**

```python
forward_backward_dump_end()
```

### 5.2 MsprobeStep

**功能说明**：用于在使用高阶 API 时，自动控制每个 step 的数据采集启停。

**使用方法：**

```python
from msprobe.mindspore.common.utils import MsprobeStep
trainer.train(epochs, dataset, callbacks=[MsprobeStep(debugger)])
```

### 5.3 seed_all

**功能说明**：固定网络中的随机性，开启确定性计算。

**使用方法：**

```python
from msprobe.mindspore import seed_all
seed_all(seed=1234, mode=True)
```

## 6. 示例代码

### 6.1 静态图场景

```python
import mindspore as ms
ms.set_context(mode=ms.GRAPH_MODE, device_target="Ascend")

from msprobe.mindspore import PrecisionDebugger
debugger = PrecisionDebugger(config_path="./config.json")
debugger.start()
# 模型定义和训练代码
# 请勿将以上初始化流程置于模型实例化或 mindspore.communication.init 调用后
```

### 6.2 动态图场景

#### 6.2.1 未使用高阶 API（非 L2 Level）

```python
import mindspore as ms
ms.set_context(mode=ms.PYNATIVE_MODE, device_target="Ascend")

from msprobe.mindspore import PrecisionDebugger
debugger = PrecisionDebugger(config_path="./config.json")

model = Network()
for data, label in data_loader:
    debugger.start(model)  # L0 Level 或采集 primitive op 时传入 model
    # 训练逻辑
    loss = model(data)
    loss.backward()
    optimizer.step()
    debugger.stop()
    debugger.step()
```

#### 6.2.2 使用高阶 API（非 L2 Level）

```python
import mindspore as ms
from mindspore.train import Model
ms.set_context(mode=ms.PYNATIVE_MODE, device_target="Ascend")

from msprobe.mindspore import PrecisionDebugger
from msprobe.mindspore.common.utils import MsprobeStep
debugger = PrecisionDebugger(config_path="./config.json")

model = Network()
trainer = Model(model, loss_fn=loss_fn, optimizer=optimizer)
trainer.train(epochs, train_dataset, callbacks=[MsprobeStep(debugger)])
```

## 7. dump 结果文件介绍

### 7.1 静态图场景

训练结束后，数据将保存在 `dump_path` 指定的目录下，目录结构请参见 MindSpore 官方文档中的[数据对象目录](https://www.mindspore.cn/tutorials/experts/zh-CN/r2.3.1/debug/dump.html)。

### 7.2 动态图场景

dump 结果目录结构示例如下：

```lua
dump_path/
├── step0/
│   ├── rank0/
│   │   ├── dump_tensor_data/
│   │   │   ├── Tensor.__add__.0.forward.output.0.npy
│   │   │   ├── ...
│   │   ├── dump.json
│   │   ├── stack.json
│   │   └── construct.json
│   ├── rank1/
│   │   ├── ...
├── step1/
│   ├── ...
```

* `rank`：设备 ID，每张卡的数据保存在对应的 `rank` 目录下。
* `dump_tensor_data`：保存采集到的张量数据。
* `dump.json`：记录算子或 API 的统计信息。
* `stack.json`：算子调用栈信息。
* `construct.json`：分层分级结构信息。

## 8. 附录

### 8.1 修改 API 支持列表

当需要改变动态图 API 级 dump 时的支持列表，可以手动修改 `msprobe/mindspore/dump/hook_cell/support_wrap_ops.yaml` 文件。例如：

```yaml
ops:
  - adaptive_avg_pool1d
  - adaptive_avg_pool2d
  - adaptive_avg_pool3d
```
