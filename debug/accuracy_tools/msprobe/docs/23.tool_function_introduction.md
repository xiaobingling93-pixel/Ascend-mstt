# msprobe 工具功能模块简介、适用场景和当前版本局限性

## 1 PyTorch框架

| <div style="width:120px"> 功能名 </div>            | <div style="width:120px"> 简介 </div>                           | <div style="width:150px"> 适用场景/优势 </div>                               | <div style="width:100px"> 当前版本局限性 </div>                                                     |
|-------------------------------------------------|---------------------------------------------------------------|------------------------------------------------------------------------|----------------------------------------------------------------------------------------------|
| [数据采集](./05.data_dump_PyTorch.md)               | 采集模型训练过程中的API或Module层级的前反向输入输出数据，包括层次关系、统计值信息、真实数据和调用栈等。      | 1、将模型中训练的API或Module的前反向输入输出数据保存下来分析<br> 2、模型出现溢出时，可用于定位首次溢出的API或Module | 1、API级数据采集仅支持白名单列表上的API<br>2、引入工具在某些场景可能会导致问题消失<br>3、当前对inplace操作API或Module的支持度有限            |
| [离线预检](./07.accuracy_checker_PyTorch.md)        | 为网络中每个API创建用例，检验其精度，并根据不同比对算法综合判定API在NPU上的精度是否达标，快速找出精度差异API。 | 1、对模型中所有的API做精度初步排查<br>2、精度排查不受模型累计误差影响                                | 1、依赖GPU环境<br>2、不支持通信算子<br>3、仅支持部分融合算子                                                        | 
| [整网比对](./10.accuracy_compare_PyTorch.md)        | 计算模型整网NPU和标杆设备的精度误差指标，标记精度异常API或Module，助力快速定位精度问题根因。          | 1、整网比对定位精度可疑算子                                                         | 1、由于使用整网dump数据，定位的可疑算子受累计误差影响<br>2、当模型规模较大时，比对所需时间较长                                         | 
| [在线预检](./08.accuracy_checker_online_PyTorch.md) | 通过TCP通信或共享存储空间的方式，进行在线精度预检，解决离线预检大数据量落盘、传输困难痛点。               | 1、使用离线预检，数据量较大落盘困难或传输耗时长时，可通过在线预检进行精度排查                                | 1、依赖GPU环境，NPU和GPU能够通信<br>2、重计算模式下，不支持反向aten算子预检                                              | 
| [溢出检查](./12.overflow_check_PyTorch.md)          | 分溢出采集和溢出数据解析两部分，助力快速定位首次溢出API或Module。                         | 1、当模型出现首次溢出时，用于快速定位首次溢出的API或Module。<br>2、相比数据采集，性能更优，磁盘压力更小            | 1、局限性同数据采集                                                                                   | 
| [数据解析](./14.data_parse_PyTorch.md)              | 互交式界面处理解析kernel层级dump数据，便于查看分析。                               | 1、比对kernel层级dump数据的一致性                                                 | 1、仅限于NPU                                                                                     | 
| [无标杆比对](./15.free_benchmarking_PyTorch.md)      | 不依赖标杆数据，通过对算子输入增加微小扰动，计算扰动后输出与原始输出的相对误差，识别有精度风险算子。            | 1、无标杆数据场景下的算子精度排查<br>2、对个别算子进行升精度、tocpu等操作，以验证其对模型loss的影响              | 1、由于需要复制输入二次执行，在遇到大张量的输入时容易显存OOM，反向显存更容易溢出，建议结合白名单使用<br>2、比对会延长训练时间，整网比对造成时长膨胀5倍以上，建议结合白名单使用 |
| [梯度状态监测](./17.grad_probe.md)                    | 可导出模型权重梯度数据并对比相似度，助力确认训练过程精度问题step和反向中的异常。                    | 1、需要分析梯度数据时<br>2、需要定位发生问题的step时                                        | 无                                                                                            | 
| [在线精度比对](./18.online_dispatch.md)               | 训练过程中直接完成NPU和CPU的精度比对并输出比对结果。                                 | 1、执行一次就可获取NPU和CPU分别执行后的精度比对结果                                          | 无                                                                                            |
| [训练状态监控](./19.monitor.md)                       | 收集模型训练过程中的激活值、梯度和优化器状态，助力分析计算、通信、优化器各部分异常情况。                  | 1、通过监控模块级统计量指标，快速定位异常模块位置，如loss出现nan                                   | 1、仅支持模块级别统计量指标分析<br>2、仅支持megatron、deepspeed框架<br>3、少量增加时间和显存膨胀                               |
| [可视化比对](./21.visualization_PyTorch.md)          | 解析dump的精度数据，还原模型图结构，比对各层级精度数据，助力理解模型结构、分析精度问题。                | 1、整网精度比对定位可疑算子，通过浏览器展示比对结果，支持快速搜索到可疑算子<br>2、支持查看模型层级结果，比对模型层级结构差异      | 1、由于使用整网dump数据，定位的可疑算子受累计误差影响<br>2、当模型规模较大时，比对所需时间较长                                         |

## 2 MindSpore框架

| <div style="width:120px"> 功能名 </div>         | <div style="width:120px"> 简介 </div>                               | <div style="width:150px"> 适用场景/优势 </div>                                                   | <div style="width:100px"> 当前版本局限性 </div>                                                     |
|----------------------------------------------|-------------------------------------------------------------------|--------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------|
| [数据采集](./06.data_dump_MindSpore.md)          | 采集模型训练过程中的API或Cell层级的前反向输入输出数据，包括层次关系、统计值信息、真实数据和调用栈等。            | 1、将模型中训练的API或Module的前反向输入输出数据保存下来分析<br> 2、模型出现溢出时，可用于定位首次溢出的API或Module<br>3、从粗粒度到细粒度进行精度定位 | 1、API级数据采集仅支持白名单列表上的API<br>2、暂未支持参数及参数梯度的采集                                                  |
| [离线预检](./09.accuracy_checker_MindSpore.md)   | 为网络中每个API创建用例，检验其精度，并根据不同比对算法综合判定API在NPU上的精度是否达标，快速找出精度差异API。     | 1、对模型中所有的API做精度初步排查<br>2、精度排查不受模型累计误差影响                                                    | 1、仅针对MindSpore.mint API                                                                      |
| [整网比对](./11.accuracy_compare_MindSpore.md)   | NPU精度数据与标杆数据的比对，支持MindSpore框架内和与PyTorch跨框架的比对，助力快速定位精度异常API或Cell。 | 1、MindSpore同框架静态图比对<br>2、MindSpore同框架动态图比对<br>3、MindSpore vs PyTorch跨框架动态图比对               | 1、部分PyTorch的API关联不到MindSpore，需要手动配置映射关系                                                      |
| [溢出检查](./13.overflow_check_MindSpore.md)     | 检测模型算子的输入输出，并在溢出时落盘数据，助力用户快速定位溢出位置。                               | 1、当模型出现溢出时，可用于定位最先溢出的API或Module<br>2、相比数据采集，性能更优，磁盘压力更小                                    | 1、API级数据采集仅支持白名单列表上的API<br>2、引入工具在某些场景可能会导致问题消失<br>3、当前对inplace操作API或Module的支持度有限            |
| [无标杆比对](./16.free_benchmarking_MindSpore.md) | 不依赖标杆数据，通过对算子输入增加微小扰动，计算扰动后输出与原始输出的相对误差，识别有精度风险算子。                | 1、无标杆数据场景下的算子精度排查<br>2、对个别算子进行升精度、tocpu等操作，以验证其对模型loss的影响                                  | 1、由于需要复制输入二次执行，在遇到大张量的输入时容易显存OOM，反向显存更容易溢出，建议结合白名单使用<br>2、比对会延长训练时间，整网比对造成时长膨胀5倍以上，建议结合白名单使用 |
